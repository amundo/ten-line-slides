Hey, it works! And it even “wraps”: if you hit the right arrow when you’re on slide 5, you go to slide 1. You might want to open up your console and watch the DOM changing as you hit the right arrow key. You’ll see the section tags rotating.

But why doesn’t the color of the slide change? That because of our CSS, which was defined relative to the order of each slide within its parent, using the :nth-child pseudo-selector. Since we’re actually changing the order of elements within the DOM, only the case where the first child is selected will actually apply. So, any CSS which is specific to a particular element in the DOM shouldn’t be applied via :nth-child.

One very simple solution would be to simply inline the background colors in the section tags themselves. The HTML will look something like this: 

  <section style=background-color:hsla(115,27%,56%,1)>first slide</section>
  <section style=background-color:hsla(42,83%,70%,1)>second slide</section>
  <section style=background-color:hsla(17,89%,72%,1)>third slide</section>
  <section style=background-color:hsla(158,20%,43%,1)>fourth slide</section>
  <section style=background-color:hsla(338,37%,54%,1)>fifth slide</section>
